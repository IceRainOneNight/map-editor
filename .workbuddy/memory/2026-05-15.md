## 黑屏修复 + 底图图层系统重构（2026-05-15）

### 修复的黑屏根因
1. **editStore.clearDrawCoords 附带重置 drawState='idle'**：画线/面时调用 clearDrawCoords 导致 draw-temp 渲染 effect 跳过渲染
2. **layers 作为 useEffect 依赖全量重建图层**：每画一个点都移除所有 layer-* 和 draw-temp 源再重建，与 draw-temp effect 竞争冲突
3. **画图时无激活数据图层**：无数据图层时画图操作静默失败

### 修复方案
- **editStore.ts**：拆分 clearDrawCoords（只清坐标）和 cancelDraw（清坐标+回到idle）；新增 insertDrawCoordBeforeLast 原子操作替代 clear+setTimeout 竞态
- **MapView.tsx 图层渲染**：改为增量更新（getSource+setData），只清理 layer-* 源，不动 draw-temp/select-highlight
- **自动创建图层**：画点/线/面时若无激活数据图层，自动创建"绘制图层"

### 底图图层系统重构
- **新建 basemapStore.ts**：BasemapLayerItem[]，支持 add/remove/toggle/opacity/reorder，默认一个高德标准底图
- **MapView 多底图叠加**：从 basemapStore 读取，按 order 堆叠，raster-opacity 控制透明度
- **LayerPanel 双区域**：上方"底图图层"（BasemapLayerItem × 添加按钮）+ 下方"数据图层"（LayerItem）
- **新建 BasemapLayerItem.tsx**：可见性切换、名称、透明度滑块、删除按钮
- **清理旧代码**：删除 mapStore.ts、BasemapSwitcher.tsx，更新 Toolbar.tsx

## 线/面不可见修复 + 绘制自动新建图层 + 图层属性编辑（2026-05-15）

### 修复：线/面绘制后不可见（stale ref 问题）
- **根因**：`finishDrawing()` 使用 `layersRef.current` / `activeLayerRef.current` 获取图层，但 `ensureDrawLayer()` 通过 `useLayerStore.getState()` 直接操作 store 创建新图层后，React ref 尚未通过重渲染更新，导致 finishDrawing 找不到新图层
- **修复**：`finishDrawing()`、`ensureDrawLayer()`、`onMapClick` 选择模式、`onKeyDown` Delete 处理器全部改用 `useLayerStore.getState()` 直接读 store，不再依赖 ref

### 新增：切换绘制工具自动新建图层
- MapView 添加 `useEffect` 监听 `tool` 变化，切换到 point/line/polygon 时自动调用 `addLayer` 创建以工具名命名的图层（如"线图层-1234"），并设为激活
- 同工具切换不重复创建（prevToolRef 去重）

### 新增：图层属性编辑（颜色、线型）
- **类型**：Layer 新增 `lineStyle: 'solid' | 'dashed' | 'dash-dot'` 字段，`LayerCreateParams` 和 `LayerUpdateParams` 同步更新，导出 `LINE_STYLE_DASHARRAY` 映射和 `LINE_STYLE_LABELS` 标签
- **Store**：`layerStore` 新增 `updateLayerProperties(id, params)` 方法，支持更新 color/opacity/lineStyle/name
- **MapLibre 渲染**：图层渲染时根据 `layer.lineStyle` 设置 `line-dasharray`（solid→空, dashed→[4,2], dash-dot→[4,2,1,2]）
- **UI**：LayerItem 激活时展开属性面板，包含颜色选择器（native color input + hex显示）和线型选择器（按钮组，SVG 预览 + 标签）
- **额外**：LayerItem 支持双击图层名编辑重命名

## 四合一修复：黑屏+GeoJSON加载+属性面板重构+GeoJSON拆分（2026-05-15）

### 1. 修复线工具黑屏问题（3处修复）
- **`editStore.ts`**：`setTool` 在 line↔polygon 切换时始终重置 drawCoords/drawState（之前未重置导致状态残留）
- **`MapView.tsx`**：数据图层 else 分支（新建源+图层）包裹 `try/catch`，避免 addSource/addLayer 失败时 JS 错误传播导致地图崩溃
- **`App.tsx`**：新增 `ErrorBoundary` 类组件包裹 MapView，崩溃时显示错误信息+重新加载按钮

### 2. 修复 GeoJSON 加载
- **`fileLoader.ts`**：`parseGeoJSON` 现在支持 `FeatureCollection` / `Feature` / 裸 `Geometry` 三种顶层类型
  - `Feature` → 自动包裹为 FeatureCollection
  - 裸 `Geometry` → 包裹为 Feature 再 FeatureCollection

### 3. 属性面板重构（LayerItem → AttributePanel）
- **Layer 类型扩展**：新增 `pointRadius`(默认5)、`lineWidth`(默认2.5)、`fillOpacity`(默认0.3) 字段
- **`LayerCreateParams`/`LayerUpdateParams`**：同步新增 pointRadius/lineWidth/fillOpacity
- **`MapView.tsx`**：渲染时使用 layer.pointRadius/lineWidth/fillOpacity 替代硬编码值
- **`AttributePanel.tsx`**：完全重写，三模式切换：
  - `empty`：无选中 → "点击地图上的要素查看属性"
  - `feature`：有选中要素 → 要素属性编辑（原功能）
  - `layer`：有激活图层无选中要素 → 图层属性编辑（颜色、透明度、点半径、线宽/线型、面填充透明度）
  - 自动检测图层几何类型（Point/LineString/Polygon），只显示相关属性区段
- **`LayerItem.tsx`**：移除内嵌属性编辑面板，恢复为纯列表项（可见性/重命名/删除）

### 4. GeoJSON 拆分（每 feature = 独立图层）
- **`layerStore.ts`** `addLayerFromFile`：遍历每个 feature 创建独立图层
  - 图层命名：`文件名-featureName`（优先取 properties.name/title/id），单 feature 时用原始文件名
  - 每个图层有独立颜色（nextColor()）、独立数据（FeatureCollection 单 feature）
